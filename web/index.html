1 · Analyze a single clip

FPS, duplicate frames and tears — computed in your browser, nothing uploaded.

How to read these numbers
  • Avg FPS — mean count of unique frames per second.
  • 1% Low — average FPS of the worst 1% of frames; the stutter indicator.
  • Unique / Duplicates — distinct frames vs. frames that just repeated the previous one.
  • Tears — horizontal frame tears (top from one frame, bottom from the next).
  • P99 ms — 99% of frames were shown faster than this frametime.

Browser demo limits

                
This page is a quick-look vitrine. For the rest, use the CLI or the Tauri desktop GUI.

            

            
    
                
  • Decode path — browser seek at a fixed 30 fps. Every metric (duplicates, tears, SSIM/MAD, sharpness) is correct on this; the only thing approximated is the container’s true frame rate / VFR.
    • 
                
  • Honest source PTS / VFR — planned next step via libav demux + WebCodecs. Not shipped yet because a streaming WebCodecs path that can still fall back to seek needs a probe/reopen design that must be validated in a browser, not written blind. The bundled libav build is kept as the demuxer for it (it has no software video decoder, so a pure-libav decode is impossible here).
    • 
                
  • Native-resolution analysis — the demo downscales (360p, or the smaller source in round-trip) to fit browser memory. Pixel-exact work on a 4K file is CLI/GUI only.
    • 
                
  • Clip length — capped at 10s (seek-based extraction is the slow part, not memory, after the streaming rewrite).
    • 
                
  • Overlay video export — not in the browser; the CLI/Tauri render frames with the FPS overlay.
    • 
                
  • Effective bitrate shown in compare is filesize ÷ duration of the downloaded file — a ranking proxy, not the host’s true encoder bitrate.
    •
